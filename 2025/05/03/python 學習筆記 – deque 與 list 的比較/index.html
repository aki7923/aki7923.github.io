使用 List 的 BFS 寫法

問題討論

使用 deque 與 list 執行BFS的差異：
雖然兩種方式在結果上沒有差異，都能正確完成 BFS，但它們在效率上會產生差異。

首先，BFS 的操作需要不斷從佇列頭部取出節點，再把新節點加到尾端。
因為 deque 是一種可以在頭尾皆可加減元素的雙向佇列。若使用 deque 的 popleft() 和 append()，兩者的時間複雜度都是 O(1)，

相對來說，因為 list 的頭是固定的，不能任意改位置，所以每次從頭拿掉一個元素時，後面所有元素都要往前移一格。這樣執行 pop() 的時間複雜度為 O(n)。

比較兩者的時間複雜度能看出，deque 的 O(1) 會快於 list 的 O(n)。
所以總結來說，雖然兩個程式都能跑出一樣的答案，但從時間效率來看，deque 比 list 更適合用來寫 BFS，特別是在資料量大的時候差距會很明顯。
